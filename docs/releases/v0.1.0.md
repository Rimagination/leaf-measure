# leaf-measure v0.1.0

First public preview of `leaf-measure`, a FAMeLeS-based leaf morphology measurement workflow packaged as both a repository and an installable skill.

## What This Release Includes

- A canonical installable skill at `skills/leaf-measure`
- Direct GitHub installation support for the standalone skill
- Shared Python CLI execution through `python -m engine.cli analyze`
- Automatic first-run bootstrap for Python dependencies, Fiji, and upstream Figshare assets
- Two supported analysis modes:
  - `Full image`
  - `Thumbnails`
- Structured outputs for agent consumption:
  - `results.csv`
  - `manifest.json`
  - `run_summary.md`
  - `method_summary.md`
  - binary and outline images
- Automatic repair paths for scanner border and polarity-related artifacts
- Machine-readable runtime diagnostics and failure reports

## What Was Validated

- Full test suite passed at release time: `30 passed`
- Direct GitHub installation path was validated end to end on Windows:
  - install `skills/leaf-measure` from `Rimagination/leaf-measure`
  - bootstrap the shared repo into `$CODEX_HOME/vendor/leaf-measure`
  - fetch Fiji and upstream assets automatically
  - run a real `Full image` analysis successfully

## Expected User Flow

### Standalone skill install

Install the skill from GitHub:

- repo: `Rimagination/leaf-measure`
- path: `skills/leaf-measure`

Then ask the agent:

```text
Use leaf-measure to analyze this folder. If I have not specified Full image or Thumbnails, explain the difference and ask me to choose. Then report area, perimeter, length, width, circularity, and solidity.
```

### Repository-local usage

If the repository is already open as the workspace, start with:

```text
Run .\scripts\bootstrap.ps1 to install Python dependencies for the current environment, download Fiji if it is missing, fetch the public Figshare assets if they are missing, and write a machine-readable doctor report.
```

## Scope And Boundaries

- Default output units are pixels
- DPI metadata is recorded when available, but physical units are not silently substituted
- The workflow is designed for published-method fidelity first, then agent usability
- `Full image` and `Thumbnails` remain distinct user-facing modes and should not be conflated
- Automatic repair only applies when specific artifact patterns are detected

## Current Limitations

- Windows is the validated platform for the full install-and-run path
- First-time setup still depends on network access for Fiji and Figshare downloads
- Segmentation quality still depends on image quality, contrast, and background conditions
- Outputs should still be visually reviewed through binary and outline artifacts

## Suggested GitHub Release Summary
